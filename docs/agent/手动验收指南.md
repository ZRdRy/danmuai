# 手动验收指南

> **原则**：`pytest` 通过或 CI 绿 **不等于** 功能可用。每个工单完成后，必须按工单中的「手动验证步骤」在真实环境走一遍关键路径。  
> 单工单记录用 [templates/手动验收/手动验收模板.md](templates/手动验收/手动验收模板.md)。发布前完整清单见 [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md)。

---

## 适用场景

| 工单类型 | 最低手动验收 |
|----------|----------------|
| 仅文档 | 链接可读、`git diff` 无业务代码 |
| Web UI | 启动应用 → 打开控制台 → 操作改动页面 |
| Overlay/弹幕 | 启动 → 启停弹幕 → 目视 Overlay |
| API/主链路 | 上述 + 日志无异常 `reason=` + 相关 pytest |

目标耗时：**5–10 分钟**（工单设计应控制在此范围内）。

---

## DanmuAI 通用启动步骤

1. **环境**（首次或依赖变更后）  
   ```bash
   pip install -r requirements.txt
   ```

2. **启动**（默认桌面壳）  
   ```bash
   python main.py
   ```  
   或使用系统浏览器：  
   ```bash
   python main.py --web-browser
   ```

3. **确认服务**  
   - Web API：`http://127.0.0.1:18765`（见 [WEB_CONSOLE.md](WEB_CONSOLE.md)）  
   - 系统托盘存在 DanmuAI 图标  
   - Overlay 窗口可显示（启停后）

4. **配置**（若工单涉及 AI）  
   - 在 Web 控制台配置有效 API Key 与模型  
   - **勿**将真实 Key 写入仓库或完成报告

---

## 通用检查项

| # | 检查项 | 预期 |
|---|--------|------|
| 1 | 应用启动 | 无崩溃；控制台可打开 |
| 2 | 启停弹幕 | 托盘/Web 可启停；状态一致 |
| 3 | Overlay | 启停后弹幕区域行为符合预期 |
| 4 | 日志 | 无意外连续错误；关注 `reason=`（见 [AGENTS.md](../AGENTS.md) 附录） |
| 5 | 退出 | 托盘退出后进程结束，无僵尸线程 |

工单特定步骤写在工单正文，**优先于**本表。

## 自动化检查（代码工单）

在手动验收**之外**，按 [AGENTS.md](../AGENTS.md) §7 运行：

```bash
python -m pytest tests/ -q
```

触达 `main.py` 编排、`app/web_api/`、`DanmuApp` 状态时：

```bash
python scripts/boundary_guard.py
```

## 不通过时

1. 在 [templates/手动验收/手动验收模板.md](templates/手动验收/手动验收模板.md) 中记录「实际结果」

3. 若属范围外缺陷 → [已知问题与后续事项.md](已知问题与后续事项.md)  
4. 若属本工单缺陷 → 同一工单返工或开修复子工单
