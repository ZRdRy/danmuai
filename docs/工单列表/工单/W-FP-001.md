## 工单 ID

W-FP-001

## 工单标题

新增 `display_mode` 总开关与悬浮窗 6 项基础配置字段

## 背景

新增「弹幕姬式」悬浮窗显示模式，需引入顶层开关 `display_mode`（overlay / floating_panel / both）与悬浮窗自身配置。
按 AGENTS.md §4「文档与代码分工」和「禁止行为」，配置字段的引入与持久化是独立的可交付单元，须先于组件本体（W-FP-002）和接入点（W-FP-003）落地。

## 目标

完成后在 `ConfigStore` 中存在以下键，`/api/config` 暴露给 Web 端可读可写：
- `display_mode`（str，枚举 overlay / floating_panel / both，默认 overlay）
- `floating_panel_opacity`（int 0-100，默认 85）
- `floating_panel_font_size`（int 12-48，默认 18）
- `floating_panel_max_items`（int 5-200，默认 60）
- `floating_panel_speed`（float 0.5-5.0，默认 1.5）
- `floating_panel_click_through`（int 0/1，默认 1）

## 需求

1. 在 `CONFIG_DEFAULTS` 字典中追加上述 6 个键的字符串默认值。
2. 在 `WEB_CONFIG_KEYS` 元组中按字母序追加同样 6 个键。
3. `display_mode` 字段在 `_normalize_items` 中校验：仅允许 `overlay` / `floating_panel` / `both`，非法值回落为 `overlay`。
4. 4 个 int 字段在 `_normalize_items` 中用 `_clamp_int_key` 钳位。
5. `floating_panel_speed` 用 float 钳位（0.5-5.0），保留 3 位小数。
6. `floating_panel_click_through` 接受 `1`/`0`/`true`/`false`，归一为 `1`/`0`。
7. `seed_config_defaults` 调用对已存在用户**不覆盖**（沿用既有 `config.get(key, "")` 判定空字符串）。
8. 单测覆盖：
   - 6 个键均在 `CONFIG_DEFAULTS` 与 `WEB_CONFIG_KEYS`；
   - 缺省状态下 6 个键均落库为默认值；
   - `display_mode="invalid"` 经 `_clamp_choice` 后落库为 `"overlay"`；
   - `display_mode="BOTH"` 归一为 `"both"`；
   - `floating_panel_max_items=9999` 被钳到 200；
   - `floating_panel_font_size=1` 被钳到 12；
   - `floating_panel_opacity=0` 允许（边界）。

## 手动验证步骤

1. 删除 `%APPDATA%/DanmuAI/config.db`（如存在）后启动 `python main.py`；
2. 关闭应用，检查 `config.db` 中 `config` 表存在 6 个新键且 `display_mode="overlay"`；
3. 启动 Web 控制台，DevTools network 观察 `GET /api/config` 响应 JSON 含 `display_mode` 字段；
4. 手工 `curl -X PUT http://127.0.0.1:18765/api/config -H 'Content-Type: application/json' -d '{"display_mode":"invalid"}'` 验证后端规范化（响应 JSON 中 `display_mode == "overlay"`）。

## 风险点

- 老用户 config.db 缺键时会被 `seed_config_defaults` 补全默认值，行为可预期；
- `display_mode` 默认 `overlay` 保证向后兼容，旧 Web UI 不感知该字段也无副作用；
- 字段落库后无法回滚（无 schema 迁移），后续若需改名由后续工单处理。
