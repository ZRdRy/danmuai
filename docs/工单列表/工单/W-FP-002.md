## 工单标题

新建 `app/floating_panel.py` 弹幕姬式悬浮窗组件本体

## 背景

W-FP-001 已落地 `display_mode` 与 6 项配置。本工单实现悬浮窗本体：独立 QWidget、置顶、可拖动、可缩放、透明度/字体/最大条数/速度/鼠标穿透均可配，**事件驱动渲染**（无持续 16ms timer）。
复用 `DanmuOverlay._apply_win32_click_through` 的 Win32 透明置顶模式（`app/overlay.py:156-168`），不与 `DanmuEngine` 共享状态。

## 目标

完成后 `from app.floating_panel import FloatingPanel` 可在主线程实例化：
- 调用 `feed(content, persona)` 追加一条弹幕；
- `display_mode in ("floating_panel", "both")` 时 `show()`，否则 `hide()`；
- 默认右下角 400×600，可拖动客户区（关闭鼠标穿透时）与右下角 `QSizeGrip` 缩放；
- 6 项配置均生效（运行时通过 `apply_config()` 重新读取）；
- 无内容时 QTimer 已停止，CPU 占用接近 0。

## 需求

1. 新文件 `app/floating_panel.py` 提供 `FloatingPanel(QWidget)`：
   - `__init__(self, config: ConfigStore)`：构造默认隐藏，绑定配置中心；
   - `feed(self, content: str, persona: str = "")`：按 `floating_panel_max_items` 上限裁剪（FIFO 丢最旧），入队后 `_kick_render()`；
   - `apply_config(self)`：从 `config` 读 6 项配置，热更新字体/透明度/最大条数/速度/鼠标穿透；
   - `set_display_mode(self, mode: str)`：仅 `floating_panel` / `both` 时 `show()`，否则 `hide()` + 停 timer；
   - `is_render_active()` / `active_count()`：辅助测试断言事件驱动行为；
   - `closeEvent` / `hideEvent` 停止 timer。
2. 动画模型：
   - `_active_items: list[_FloatingItem]`，每条 `y -= speed * dt * 60`（向上滚动，与 `DanmuEngine` 的 `speed` 语义对齐）；
   - `_tick()` 每 16ms：推进位置 → 移除 `y + height < 0` 的条目 → 若列表非空继续，否则 `timer.stop()`；
   - `_kick_render()` 调用 `self.timer.start(16)`，已 active 则跳过；
   - 帧间间隔用 `QElapsedTimer` 实测，限制 dt ≤ 100ms。
3. 渲染：
   - `paintEvent` 遍历 `_active_items` 调用 `_item_opacity(item) * global_alpha` 绘制对应 pixmap；
   - 字体大小 = `floating_panel_font_size`；全局 alpha = `floating_panel_opacity / 100`；
   - 顶 30px（`_FADE_OUT_PX`）做 alpha 渐隐，底 30px（`_FADE_IN_PX`）做 alpha 渐入；与 `DanmuOverlay` 的 `FADE_OUT_PX` / `FADE_IN_PX`（`app/danmu_engine.py:41-43`）对称。
4. 交互：
   - `mousePressEvent` / `mouseMoveEvent` / `mouseReleaseEvent` 实现客户区拖动（`WindowStaysOnTopHint | Tool`，**不**设 `FramelessWindowHint`）；
   - 右下角嵌入 `QSizeGrip` 提供缩放手柄；
   - 仅当 `_click_through is False` 时响应鼠标拖动（透明穿透时不抢事件）。
5. Win32 透明置顶（仅 Windows）：
   - `show()` 后调用 `_apply_win32_click_through`，OR `WS_EX_LAYERED`，根据 `_click_through` 决定是否 OR `WS_EX_TRANSPARENT`；
   - 与 `DanmuOverlay._apply_win32_click_through` 实现完全独立（**不**跨文件 import 私有方法），`app/floating_panel.py` 自带常量与 ctypes 调用。
6. 单测 `tests/test_floating_panel.py` 覆盖（基于 `QApplication` fixture）：
   - 初始 timer 非 active；
   - `feed()` 启动 timer；
   - 队列空时 `timer.isActive() == False`（**关键性能验收**）；
   - 上限裁剪（`max_items` 调整时按 FIFO 立即裁剪）；
   - `apply_config` 反映 ConfigStore 改动；
   - `set_display_mode("overlay" | "weird")` 隐藏；
   - 内部 `deque(30)` 去重窗口生效；
   - 空文本被忽略；
   - `display_mode=overlay` 时 `feed()` 直接 return；
   - `apply_config` 钳位（`font_size=999 → 48`、`opacity=200 → 100`、`max_items=0 → 5`、`speed=100 → 5.0`）。

## 非目标

- 不接入 `_consume_reply_queue`（W-FP-003 处理）；
- 不改 `DanmuApp.__init__` 之外的 `main.py` 方法；
- 不实现 persona 颜色/头像/表情（统一白色 + 黑色描边，与 `DanmuOverlay._render_item_pixmap` 类似即可）；
- 不与 `DanmuEngine.recent` 共享去重窗口（独立 `deque(30)` 内部去重）；
- 不做"全屏多列横向滚动"等变体（用户明确只要弹幕姬式单列向上）；
- 不落 `floating_panel_geometry`（位置/大小由 W-FP-002 内 `resizeEvent` 维护内存，W-FP-003 由 UI 触发 `apply_config` 时再考虑持久化）。

## 手动验证步骤

1. 启动 `python main.py`（默认 `display_mode=overlay`，悬浮窗隐藏）；
2. 临时把 `main.py:194` 之后插入 `self.floating_panel.set_display_mode("floating_panel")` 仅用于本步验证（验证后还原）；
3. `self.floating_panel.feed("hello 弹幕姬 1")` 反复调用 30 次；
4. 观察：所有条目从底部淡入 → 向上滚动 → 顶部淡出；30 条全部消失后窗口静止（**关键**）；
5. 拖动窗口到屏幕中央，缩放到 300×400，再次喂入确认位置/大小生效；
6. `config.set("floating_panel_click_through", "0")` 后 `self.floating_panel.apply_config()`，用鼠标拖动窗口；切回 `"1"` 后点击穿透到下层窗口。

## 风险点

- `setWindowFlags` 不含 `FramelessWindowHint` 时有原生标题栏；本工单保留 `WindowStaysOnTopHint | Tool`，由 `QSizeGrip` 与手写拖动协同；
- 高 DPI 屏字号 / 文字宽度已用 `devicePixelRatio()` 处理；
- Win32 `WS_EX_TRANSPARENT` 在 `WS_EX_LAYERED` 未设时无效，二者同时 OR；运行时切换 `_click_through` 通过重新 `_apply_win32_click_through` 调整 `ex_style`；
- 性能：**绝不能** `timer.start(16)` 不停——单元测试显式断言「队列空 → timer 停」。
