## 工单标题

将悬浮窗接入 `_consume_reply_queue` 旁路分发并暴露 Web 设置项

## 背景

W-FP-001 落字段、W-FP-002 实现组件。本工单做最后接线：在 `DanmuApp._consume_reply_queue`（`main.py:1284`）上屏后**旁路**通知悬浮窗（不改主链路、不重发 AI），并在 Web 控制台「弹幕显示」设置区暴露 `display_mode` 等 6 项配置。
按 AGENTS.md §9 第 1/3 条：所有 Qt 改写走 `WebConsoleBridge` 或 `QTimer.singleShot(0, ...)` 到主线程；HTTP 线程禁止直接读 `danmu_app._…`。

## 目标

完成后：
- 主链路 `_consume_reply_queue` 上屏一条弹幕后，悬浮窗（若开启）收到相同文本；
- Web 端 `GET /api/config` 返回新增 6 项字段；
- `PUT /api/config` 中修改 `display_mode` 立即生效（开/关悬浮窗），无需重启；
- 修改 `floating_panel_opacity` / `font_size` / `max_items` / `speed` / `click_through` 后，下次 `apply_config()` 生效。

## 需求

1. **主链路分发**：`main.py::_consume_reply_queue` 在 `if item:` 块末尾（`_broadcast_live_overlay_item` 之后）追加：
   ```python
   self._feed_floating_panel(display_content, queued.persona_id)
   ```
   新增私有方法 `_feed_floating_panel(content, persona_id)` 与 `_floating_panel_enabled()`（读 `config.get("display_mode")`，值为 `floating_panel` / `both` 时 True）；`panel.feed` 抛异常时**绝不**影响主链路（仅 debug 日志）。
2. **配置变更回调**：`_on_config_changed` 末尾（`self._sync_mic_service()` 之后）追加：
   ```python
   if panel is not None:
       panel.apply_config()
       panel.set_display_mode(self.config.get("display_mode", "overlay"))
   ```
   与主线程 signal 链路衔接（HTTP → `invoke_on_main` → `apply_web_payload` → `config_changed.emit` → 主线程 `_on_config_changed`）。
3. **Web 设置项**（`index.html`）：在「弹幕显示」tab 末尾追加 6 个字段（`<select>` × 1 + `<input type="number">` × 4 + `<input type="checkbox">` × 1），不新建 tab 减少 Web 改动。
4. **Web JS**（`settings.js`）：
   - `CONFIG_FIELDS` 追加 6 键；
   - `SETTINGS_RESTORE_GROUPS.danmu` 追加 6 键（参与「恢复默认」）；
   - `applyDefaultToField` 对 `floating_panel_click_through` 做 checkbox 处理；
   - `fillForm` 用 `setIfEmpty` 兜底 5 个非 checkbox 字段；单独处理 `floating_panel_click_through`；
   - `collectFormData` 把 `floating_panel_click_through` 序列化为 `'1' / '0'`；
   - `display_mode` 走 enum 校验（`overlay` / `floating_panel` / `both`，非法值回落 overlay）；
   - `SETTINGS_HELP_TEXT` 追加 6 条 help 文案。
5. **端到端测试**：
   - `tests/test_floating_panel.py` 追加 5 个用例：`_floating_panel_enabled` 判定、`feed_floating_panel` routing / overlay-mode skip / 异常吞掉、`_consume_reply_queue` 调用 spy 验证；

## 风险点

- `_consume_reply_queue` 的改动**最小化**：仅追加 1 行分发，**绝不**改既有 `if item:` 块语义；
- `apply_config_patch` 是主线程方法，被 HTTP 通过 `invoke_on_main` 调用；**严禁**在 HTTP 线程直接调 `self.floating_panel.*`；
- `_feed_floating_panel` 异常吞掉是为了**绝不**让悬浮窗问题影响主链路；调试用 `debug` 级日志，不污染生产日志；
- 边界：若用户在悬浮窗模式下用 `python main.py --web-browser` 启动，pywebview 主窗仍存在，悬浮窗是**独立**置顶窗，与主窗互不遮挡。
