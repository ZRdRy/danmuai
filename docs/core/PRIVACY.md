# Privacy

## 截图与传输

- 默认会截取**配置的目标显示器全屏**（`screen_index`，由 `app/snipper.py` 的 `ScreenCapturer` 解析；无效索引回退 0）。
- 当 `region_w` / `region_h` 大于 0 时，会按所选屏幕左上角的相对坐标使用 `region_x` / `region_y` / `region_w` / `region_h` 裁剪截图；无效区域回退全屏，越界区域会夹取到屏幕范围内。
- 截图在内存中压缩（PIL + JPEG + Base64）后，发送到你配置的 AI 服务端点，用于生成弹幕文本。
- 截图**默认不落盘**；程序默认只保存弹幕文本历史。

## 麦克风模式（实验）

- 在 Web 设置中开启「麦克风模式」后，程序会通过 `sounddevice` 在**内存**中缓冲麦克风 PCM，**默认不写入磁盘**。
- 检测到**说话结束**时，会**独立于截图节奏**额外发起一次 AI 请求（与当前截图一并发送），生成接话弹幕并插队显示；截图轨道的 API 调度不受影响。
- 每次插入请求附带最近若干秒 WAV Base64（`data:audio/wav;base64,...`），需**豆包 Responses** 与支持**音频理解**的模型。
- 说话频繁可能增加 API 调用次数（内置冷却与最短语音时长限制）。
- 麦克风音频与截图一样会离开本机并进入你所选 AI 服务商；请确认环境中无敏感对话。
- 未开启麦克风模式时，不进行麦克风采集。

## 本地存储

- 配置库默认位于 `%APPDATA%/DanmuAI/config.db`。
- API Key 优先使用 Fernet 加密；无 `cryptography` 依赖时会退化为 base64 编码，并在日志中明确提示风险。
- 仓库中的 `log/`、`ph/`、`.coverage`、缓存目录均属于本地调试产物，发布前应忽略，不应进入版本控制。

## 日志策略

- 日志会脱敏 API Key、Bearer Token、长 base64 图片与音频和加密串。
- 日志不会记录完整截图内容，也不会记录原始请求体。

## 使用建议

- 使用前确认**目标显示器**上没有敏感窗口（账号、密码、聊天、支付、隐私文档等）。
- 直播、录屏或演示时，尽量将敏感应用移到非目标屏或最小化；若使用局部区域，请确认裁剪区域内也没有敏感内容。
- 可视化选区仍在 [ROADMAP.md](ROADMAP.md) 中规划；当前版本需要手动填写 region 坐标。
