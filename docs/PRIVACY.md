# Privacy

## 截图与传输

- DanmuAI 只会截取你配置的区域，不做默认全屏截图。
- 截图会发送到你配置的 AI 服务端点，用于生成弹幕文本。
- 截图默认不落盘，程序默认只保存弹幕文本历史。

## 本地存储

- 配置库默认位于 `%APPDATA%/DanmuAI/config.db`。
- API Key 优先使用 Fernet 加密；无 `cryptography` 依赖时会退化为 base64 编码，并在日志中明确提示风险。
- 仓库中的 `log/`、`ph/`、`.coverage`、缓存目录均属于本地调试产物，发布前应忽略，不应进入版本控制。

## 日志策略

- 日志会脱敏 API Key、Bearer Token、长 base64 图片和加密串。
- 日志不会记录完整截图内容，也不会记录原始请求体。

## 使用建议

- 只截取必要区域。
- 避免选择包含账号、密码、聊天、支付、隐私文档的区域。
- 如果你在直播、录屏或演示场景中使用，请先确认选区不会覆盖敏感窗口。
