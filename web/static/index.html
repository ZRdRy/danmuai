DanmuAI

能做什么：设置 AI 接口、用哪块屏幕识图、弹幕样式，以及截图与压缩参数。改完后点「保存配置」立即生效。

五个分页：API 与模型、识图区域、弹幕显示、节奏与图像、读弹幕。想改人设和提示词请去「人格工坊」。

使用前注意：请先在「温馨控制台」开始或停止弹幕；API 密钥留空保存不会改掉旧密钥；默认识图所选显示器全屏，可在「识图区域」鼠标框选子区域；「模型配置档案」可单独填地址和密钥。

模型配置档案

麦克风模式

开启后，说完一句话会额外生成几条接话弹幕，并插队显示，不影响看屏识图的节奏。请说完后停顿约半秒；环境较吵时启动后会自动校准。录音只在内存里处理，不会保存到硬盘。

默认开启：开麦使用「API 与模型」里的接口与视觉模型。关闭后可单独配置支持麦克风的模型（识图仍用视觉模型）。

仅列出目录中带「支持麦克风」标识的模型；豆包全模态或小米 MiMo 的 mimo-v2.5。

「测试麦克风」只检查有没有声音，不会联网；「测试发送」会录音并请求 AI。请先点「保存配置」再测。用蓝牙耳机时，请在 Windows「设置 → 系统 → 声音 → 输入」里把耳机设为默认麦克风。

识图与隐私区域

默认识图所选显示器全屏；也可点击下方按钮用鼠标框选子区域（坐标相对该屏左上角）。请在「API 与模型」选好识图显示器后再框选。截图只在内存里压缩后发给 AI，默认不会保存到硬盘。

当前识图范围

  读弹幕 
 
使用语音合成定时随机朗读当前屏幕上的一条弹幕
 
 
默认使用小米 MiMo（mimo-v2.5-tts）；也可配置 OpenAI 兼容的 chat/completions 音频接口（响应需含 message.audio.data）。TTS API Key 与助手设置中的视觉 API Key 相互独立，请在本页单独填写。
 
仅在已开始生成且屏幕上有可见弹幕时才会朗读；停止生成后不再请求。
 
MiMo 参数说明见 语音合成文档。
 
 
  
   
每隔若干秒随机选一条当前屏上可见弹幕合成语音（3–120 秒）。间隔过短会在上一句播完之前无法开始下一句；合成约需 10–20 秒，建议间隔 ≥15 秒。
 
 
 
自定义 TTS 模型
 
读弹幕是语音合成（TTS）链路，与助手设置中的聊天/视觉模型不同。OpenRouter、One API 等普通聊天模型通常无法返回音频，请勿直接复用弹幕生成模型。
 
   
 
   
 
   
 
选择「默认」时不保存自定义字段，朗读使用 MiMo 出厂配置。自定义平台须同时填写 API 地址与模型名称。
 
 
   
 
   
 
   
对应 API 中 role 为 user 的消息，用于控制语气与风格；新装默认已填泪目治愈向指令，留空则仅朗读弹幕正文。

公式化弹幕库
 
管理自定义公式化短句，用于同屏不足时自动补足
 
 
 
 
弹幕库来源
 
  
启用后，系统会从你保存的自定义弹幕句中抽取短句，用于弹幕不足时补足。
 
 
 
 
补足规则
 
   
 
当屏幕上的弹幕少于该数量时，会从自定义公式化弹幕库中抽取短句补足。
 
关闭自定义公式化弹幕库后，不会进行公式化弹幕补足。
