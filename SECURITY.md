# Security Policy

## 支持范围

当前仓库维护的是桌面端主干版本。安全修复会优先覆盖当前默认分支。

## 报告方式

- 不要在公开 Issue 中粘贴 API Key、请求头、截图原图或包含隐私内容的日志。
- 如果问题涉及凭据泄漏、截图隐私或可远程利用的漏洞，请私下联系维护者处理。

## 项目当前安全边界

- API Key 默认保存在 `%APPDATA%/DanmuAI/config.db`，优先使用 Fernet 加密。
- 日志会脱敏 API Key、`Authorization` 头、长 base64 图片数据和加密串。
- 默认不保存截图，不会把截图原文写入日志。
- 过期请求和旧场景回复会被丢弃，避免旧内容覆盖当前画面。

## 使用建议

- 只选择需要分析的局部区域，不要截取整屏。
- 避免让截图区域覆盖密码框、聊天窗口、支付页面、企业内网内容。
- 发布构建或共享代码前，确认仓库不包含 `log/`、`ph/`、本地数据库、`.key`、缓存目录。
